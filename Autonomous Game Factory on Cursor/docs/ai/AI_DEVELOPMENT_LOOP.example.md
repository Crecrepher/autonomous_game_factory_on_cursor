# AI 개발 루프

> `.example`을 제거하고 프로젝트에 맞게 수정하세요.

---

## 루프 개요

```
1. Read docs  →  2. Plan  →  3. Build  →  4. Validate  →  5. Summarize  →  6. Improve
```

---

## 단계별 상세

| 단계 | 입력 | 산출물 |
|------|------|--------|
| 1. 문서 읽기 | Docs/ai/*.md, MODULE_REGISTRY.yaml | 제약·도메인 이해 |
| 2. 계획 | 기능 요구사항 | PLAN (모듈 매핑, 파일 목록) |
| 3. 코드 생성 | PLAN, CODING_RULES | 변경/추가된 소스 |
| 4. 검증 | Tools/AI/Validate | 검증 보고서 |
| 5. 리스크 요약 | 검증 보고서 | 변경 파일, risk |
| 6. 개선 | 실패 원인 | 수정 → 4단계 회귀 |

---

## 운영 원칙

- 한 사이클 = 한 기능 / 한 모듈
- Core 수정 필요 시 PLAN에 명시, 사용자 허용 후에만
- 검증 통과 전까지 확정 아님
- 매 사이클 끝에 변경 파일 + risk 요약
