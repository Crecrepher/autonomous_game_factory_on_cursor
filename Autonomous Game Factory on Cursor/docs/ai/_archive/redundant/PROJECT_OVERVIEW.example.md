## 4. 아키텍처 철학

### 4.1 원칙

- **SOLID** — 단일 책임, 개방-폐쇄, 리스코프 치환, 인터페이스 분리, 의존성 역전.
- **의존성 주입** — 생성자/세터/ScriptableObject 등으로 주입, 하드코딩 지양.
- **이벤트 기반 통신** — 필요한 객체끼리만 통신, 전역 이벤트 버스 사용 금지.
- **느슨한 결합·높은 응집** — 모듈 간에는 인터페이스로만 의존.

### 4.2 구조적 원칙

- **MonoBehaviour는 얇게** — 로직은 Runtime 클래스로 분리.
- **네임스페이스** — 게임 제목 기준.
- **ScriptableObject** — 설정·데이터에 사용.
- **자식만 참조** — 동일·상위 오브젝트 직접 참조 금지.

## 5. 폴더 구조 설명

```
Assets/
  Game/
    Core/          # 부트스트랩, 입력, 카메라, 세이브 (AI 비편집 권장)
    Shared/        # 공용 타입, 인터페이스, 상수
    Modules/
      Template/    # 표준 모듈 템플릿
      [Module]/    # 프로젝트 모듈들
Docs/
  ai/
    PROJECT_OVERVIEW.md
    CODING_RULES.md
    MODULE_REGISTRY.yaml
```

모듈별 경로·의존성·편집 가능 여부는 `MODULE_REGISTRY.yaml`에서 관리한다.
